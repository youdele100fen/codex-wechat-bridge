# 我把 Codex 接进微信了：现在可以在手机上继续给它派活

很多人用 Codex 时，都卡在同一个地方：

电脑上任务跑起来了，但人已经不在电脑前。

结果只能靠两种很笨的方法处理：

- 要么一直坐在电脑旁边等
- 要么隔一阵回来看一次

如果任务做完了，你还想补一句“继续整理”“再改一下”“换个格式输出”，通常又得重新回到电脑前。

这也是我做 `codex-wechat-bridge` 的原因。

它想解决的不是“再造一个聊天机器人”，而是一件更具体的事：

**让 Codex 在电脑上继续工作，而你可以在微信里收结果、补下一步要求。**

项目地址：

- GitHub: <https://github.com/youdele100fen/codex-wechat-bridge>

---

## 它到底是什么

一句人话解释：

`codex-wechat-bridge` 是一个把 **Codex Desktop** 和 **微信** 连起来的开源插件。

它做两件事：

1. 电脑上的 Codex 任务做完后，把结果自动发到微信
2. 你在微信再发一句新要求，它会把这句话接回刚才那个 Codex 任务里继续执行

所以它不是那种“随便聊两句”的 AI 微信机器人。

它更像一个远程办公桥：

- 电脑端继续跑任务
- 微信端负责接收通知
- 微信端也可以继续往下派活

---

## 为什么这件事有用

因为很多真实工作场景，根本不是一直坐在电脑前完成的。

比如你可能会遇到这些情况：

- 在电脑上让 Codex 整理资料、生成摘要、检查文稿，然后出门开会
- 任务跑完时你人在路上，希望微信第一时间收到结果
- 看到结果后，你只想再补一句：“继续把上面整理成表格”
- 或者：“把刚才结果改成更正式的版本”

如果没有桥接，这些动作都要回到电脑前。

有了桥接以后，流程就变成：

- 电脑上先启动任务
- 微信里收到完成通知
- 微信里继续补下一句
- Codex 回到刚才那个任务里继续做

这才更接近“远程手机办公”。

## 先用白话解释几个词

如果你还没接触过这些概念，可以这样理解：

- **Codex Desktop**：就是你电脑上的 Codex 客户端
- **WeChat ClawBot**：就是负责收发微信消息的通道
- **codex-wechat-bridge**：就是把前面这两者连起来的插件

也就是说：

- 它不会凭空生成一个微信机器人
- 它是基于现成微信通道来工作的
- 它的重点是把“桌面 Codex 的任务流程”接到微信里

## 它现在已经能做什么

目前公开版本，已经能稳定完成两件核心事情。

### 1. 任务结果自动发到微信

你在电脑端让 Codex 做完一件事后，微信会收到一条通知。

这条通知里会清楚显示：

- 来自哪个 Project
- 来自哪个 Thread
- 这次的 Prompt 是什么
- 结果摘要是什么

这就意味着，你不用一直盯着电脑等结果。

### 2. 在微信继续发下一条 Prompt

这是这个项目最有用的地方。

假设你刚刚在微信里收到了某个 Codex 任务的完成通知。

这时你再发一句：

```text
把上面的内容整理成表格
```

桥接不会把它当成一个全新的闲聊窗口。

它会把这句话送回去，接到你刚刚那个 Codex 任务里继续往下做。

说得再直白一点：

**你是在微信里继续指挥电脑上的那个任务。**

---

## 两个最容易理解的真实场景

### 场景 1：结果通知

你在电脑端让 Codex 做一件事，比如：

```text
列出我 Mac 桌面上的所有文件名称
```

等它做完后，微信会收到类似这样的内容：

```text
Codex 任务完成通知
Project：Academic Workflow 2
Thread：桌面文件整理
Prompt：列出我 Mac 桌面上的所有文件名称
结果：......
```

这个时候，你人在外面也能第一时间知道任务结果。

### 场景 2：继续补一句要求

你在微信看到结果后，继续发一句：

```text
把上面的内容整理成一个表格
```

这条要求会继续回到刚才那个任务里，而不是重新开一个没有上下文的新聊天。

这一步很关键。

因为真正好用的，不是“微信里也能问 AI”。

而是：

**微信里的下一句，能接着电脑上的上一句继续做。**

这套方案最适合下面这些人：

- 已经在 Mac 上使用 Codex Desktop 的用户
- 想让 Codex 任务结果自动发到微信的人

- 想把手机变成 Codex 远程续接入口的人

## 也要提前说清楚：它不适合谁

如果你是下面这些情况，这个项目暂时不适合你：

- 你不用 macOS
- 你没有使用 Codex Desktop
- 你想要的是一个通用的微信 AI 聊天机器人
- 你不想处理一次性的系统授权和微信登录

因为当前版本依赖这 4 个前提：

- macOS
- Codex Desktop
- WeChat ClawBot
- 一次性的 macOS 辅助功能授权

## 最短安装路径

如果你看完已经想试，最短就是下面 5 步。

### Step 1：下载项目

```bash
git clone https://github.com/youdele100fen/codex-wechat-bridge.git
cd codex-wechat-bridge
```

### Step 2：安装命令

```bash
./install.sh
```

### Step 3：完成首次设置

```bash
codex-wechat setup
```

### Step 4：检查环境

```bash
codex-wechat doctor
```

### Step 5：启动桥接

```bash
codex-wechat start
```

如果你是第一次接触这个项目，这里有两个重点一定要记住：

1. 你需要先完成微信登录
2. 你需要给终端一次 macOS 辅助功能授权

更详细的安装和排障说明，可以直接看 GitHub 仓库里的 README 和小白教程。

## 第一次怎么验证有没有真的跑通

推荐按这个顺序测试：

1. 先打开 Codex Desktop
2. 在电脑端正常完成一个真实任务
3. 等微信收到第一条任务完成通知
4. 再从微信发一句新的 Prompt

如果都正常，你会看到：

- 微信能收到任务结果
- 你在微信里补发的新要求，会进入刚才那个任务继续执行
- 最终结果还是通过微信通知回到手机上

## 使用前一定要知道的 4 个注意事项

### 1. 它不是普通聊天机器人

它的本质是“微信和桌面 Codex 的桥接器”，不是一个独立的闲聊 bot。

### 2. 它默认接到最近一次通知对应的任务

你在微信里发的新要求，默认会接到“最近一次收到通知的那个 Codex 任务”里继续做。

### 3. 最近一次通知可能来自别的 Project

如果你最近一次收到的通知来自另一个 Project，那么微信里的下一句要求也会跟着切过去。

### 4. 想稳定使用，最关键的是这三项

- Codex Desktop 正在运行
- 微信通道登录成功
- macOS 辅助功能权限已经给好
